## 1. Introduction and Motivation 介紹與動機

### Device-to-Device (D2D) Communications 設備間通訊
- **Why D2D? 為何使用 D2D？**
  - Offloading traffic from infrastructure 減輕通訊基礎設施負載  
  - Real-time response for proximity-based applications 近距離應用的即時回應

### Controllable D2D Communications 可控的 D2D 通訊
- **Entities 參與實體**:
  - Alice, Bob, Base Station, HSS, Authentication Center  
  - 需保護身份與認證資訊

## 2. System and Security Models 系統與安全模型

### Security Threats 安全威脅
- Attacks targeting specific apps 針對特定應用的攻擊  
- Need to secure group and communication information 保護群組與通訊資訊  
- Traceability and revocability 可追蹤性與可撤銷性

### Security Requirements 安全需求
- End-to-end secure communication 端到端安全通訊  
- Group anonymity against app usage disclosure 群組匿名性  
- Accountable membership management 可追蹤的成員管理  
- Identity anonymity 身份匿名性

---

## 3. Security Definitions 安全定義

### Match Conversation 匹配對話
- Ensures both parties have matching conversation records 確保雙方對話一致  
- No matching / Multiple matching 無匹配或多重匹配問題

### Oracles for Attackers 攻擊者可使用的查詢介面
- Execute, Send, Reveal, Corrupt, Test, TestID, TestGroup

### Security Goals 安全目標
- Mutual Authentication 相互認證  
- Key Exchange Against System Operator 對抗系統運營商的密鑰交換  
- Identity Anonymity 身份匿名性  
- Group Anonymity 群組匿名性

---

## 4. Proposed Schemes 提出的方案

### Network-Covered Group Anonymous D2D Authentication 網路覆蓋下的群組匿名 D2D 認證
- UE 與 HSS/AuC 共享 ID, K  
- UE 與 ProSe 功能共享 AID, AK  
- ProSe 與 HSS/AuC 共享 AID  
- HSS/AuC 作為金鑰生成中心  
- ProSe 生成唯一密鑰 Kp 與 (pk_p, sk_p)

### Network-Absent D2D Communications 無網路 D2D 通訊
- 使用 Commitments 承諾群組資訊  
- 同群組的兩個 UE 使用相同隨機數產生相同雜湊值

### Traceability on Group Anonymity 群組匿名性的可追蹤性
- 使用 Zero-Knowledge Proofs 零知識證明  
- 比較兩個相同訊息的密文（BBS04 方法）

## 6. Security Proofs 安全性證明

### Concept 概念
- **Ideal World 理想世界**：使用理想功能（如 F_MA, F_KE, F_ID-Anon, F_G-Anon, F_Tr）  
- **Real World 真實世界**：使用實際實現（如 PRF, PRP, IBE, k-anonymity handshakes, ZKP）

### Game-based Proofs 基於遊戲的證明
- Game_0：所有轉錄由理想功能產生  
- 逐步將理想功能替換為真實功能，並分析攻擊者優勢
